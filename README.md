# ADPCM-A/B Audio Converter

Windows GUI tool for converting single WAV/MP3 files or whole folders through
NeoGeo YM2610 ADPCM-A or ADPCM-B processing.

## Run

Double click `run_converter.bat`, or run:

```powershell
python .\neogeo_adpcm_converter.py
```

## Input

- Microsoft PCM WAV files.
- MP3 files.
- Mono and stereo are accepted; stereo is mixed to mono.
- 8-bit, 16-bit, 24-bit and 32-bit integer PCM WAV data are accepted.
- ADPCM-A is resampled to 18500 Hz.
- ADPCM-B uses the selected sample rate and logs the matching Delta-N value.

## Output

The main output is an audible WAV file:

- `name_adpcma.wav`
- `name_adpcmb.wav`

These WAV files are generated by encoding to NeoGeo ADPCM-A/B and decoding back
to PCM, so they represent the ADPCM result.

From the GUI, `Output format` is a drop-down menu. The default selection is
`WAV`.

The available choices are:

- `WAV`
- `Raw NeoGeo`
- `WAV + Raw NeoGeo`

When raw NeoGeo data is enabled, the app writes:

- `name.adpcma`
- `name.adpcmb`

Raw output data is padded so every file size is aligned to the NeoGeo 256-byte
sample boundary. Each conversion run also writes `adpcm_manifest.csv` in the
output folder.

If `Use source folder and create an output subfolder` is checked, the GUI writes
to an `output` folder next to the selected source. For a selected folder it uses
`selected-folder\output`; for a selected WAV/MP3 file it uses
`file-folder\output`.
When recursive scan is enabled, that `output` folder is skipped during input
scanning.

## Controls

- `Codec`: choose ADPCM-A or ADPCM-B.
- `Preset`: quick ADPCM-B sample-rate and filter choices.
- `Sample rate`: target rate for ADPCM-B; ADPCM-A is fixed.
- `Volume dB`: post-normalization gain.
- `Max duration s`: hard trim long samples.
- `Low-pass pre-encode`: simple low-pass filter before ADPCM compression.
- `Trim leading/trailing silence`: remove quiet edges using the dBFS threshold.
- `Fade-out on trim`: fades the end after trailing trim or max-duration trim.
- `Normalize to -1 dBFS`: normalizes peak level before volume gain.
- `Recursive subfolder scan`: preserves subfolder layout under the output folder.
- `Output format`: choose `WAV`, `Raw NeoGeo`, or `WAV + Raw NeoGeo`.
- `Use source folder and create an output subfolder`: ignores the manual output
  path and writes to a source-local `output` folder.
- `Preview`: encodes and decodes the first selected audio file to a temporary WAV and
  plays it, so you hear the ADPCM result.

## Command line

```powershell
python .\neogeo_adpcm_converter.py --input .\wav --output .\out --codec ADPCM-B --sample-rate 22050 --recursive --normalize --low-pass
```

MP3 input works in the EXE build. When running from source, install
dependencies first:

```powershell
python -m pip install -r requirements.txt
```

Raw output only:

```powershell
python .\neogeo_adpcm_converter.py --input .\wav --output .\out --codec ADPCM-B --output-format raw
```

WAV and raw output:

```powershell
python .\neogeo_adpcm_converter.py --input .\wav --output .\out --codec ADPCM-B --output-format both
```

Self-test:

```powershell
python .\neogeo_adpcm_converter.py --self-test
```

## EXE build

Run `build_exe.bat` to build `dist\NeoGeoADPCMConverter.exe` with PyInstaller.
This step installs PyInstaller with pip if needed.

## Notes

The encoder logic follows public NeoGeo/YM2610 ADPCM-A and ADPCM-B references:

- https://wiki.neogeodev.org/index.php/ADPCM
- https://wiki.neogeodev.org/index.php?title=ADPCM_codecs
- https://github.com/freem/adpcma
